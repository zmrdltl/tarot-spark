# tarot-spark Agent Guide

This file is the repo-local router for human and AI-assisted contributor work.
Keep policy details in `docs/contributing/`; do not duplicate them here.

## Task Routing

- Workflow choice: read `docs/contributing/workflow.md`.
- Preflight before file edits: follow `docs/contributing/workflow.md#preflight`.
- Code changes: read `docs/contributing/code-quality.md` and `docs/contributing/quality-gates.md`.
- Product, analytics, AdSense, SEO, or tarot content changes: read `docs/contributing/product-guardrails.md`.
- Issue creation or issue planning: read `docs/contributing/issue-workflow.md`.
- Branch naming: read `docs/contributing/branch-naming.md`.
- Branch creation or branch updates: read `docs/contributing/branch-workflow.md`.
- Commit messages: read `docs/contributing/commit-messages.md`.
- Documentation, README, rule files, or public copy changes: read `docs/contributing/documentation-style.md`.
- PR creation or issue linking: read `docs/contributing/pull-requests.md`.
- PR review: read `docs/contributing/pr-review.md`.
- Merge or rebase decisions: read `docs/contributing/merge-strategy.md`.

## Planning

Plans do not belong in this file. Keep short plans in the current task, and use
GitHub Issues only when tracking, collaboration, deferred follow-up, or
acceptance criteria need a durable record.
