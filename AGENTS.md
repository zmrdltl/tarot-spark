# tarot-spark Agent Guide

This file is the repo-local router for human and AI-assisted contributor work.
Keep task details in the linked docs; do not duplicate them here.

## Task Routing

- Docs map or document placement: read `docs/README.md`.
- Workflow choice: read `docs/workflow/task-routing.md`.
- Preflight before file edits: follow `docs/workflow/task-routing.md#preflight`.
- Code changes: read `docs/engineering/code-standards.md` and
  `docs/engineering/verification-gates.md`.
- Product, analytics, AdSense, SEO, or tarot content changes: read
  `docs/product/guardrails.md`.
- Issue creation or issue planning: read `docs/workflow/issues.md`.
- Branch naming, creation, or updates: read `docs/workflow/branches.md`.
- Commit messages: read `docs/workflow/commits.md`.
- Documentation, README, rule files, or public copy changes: read
  `docs/engineering/documentation-standards.md`.
- PR creation or issue linking: read `docs/workflow/pull-requests.md`.
- PR review: read `docs/workflow/reviews.md`.
- Merge or rebase decisions: read `docs/workflow/merges.md`.

## Planning

Plans do not belong in this file. Keep short plans in the current task, and use
GitHub Issues only when tracking, collaboration, deferred follow-up, or
acceptance criteria need a durable record.
